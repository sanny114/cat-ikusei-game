## ゲーム仕様

### ステータス

すべて 0〜100 の範囲。初期値は 満腹55・清潔55・体力55・気分50。

- 満腹・清潔・体力: 画面のメーターに表示
- 気分: 隠しポイント。表示は 70以上「ごきげん」／30〜69「ふつう」／30未満「しょんぼり」

### お世話ボタンの効果

| ボタン | 効果 |
|---|---|
| ごはん | 満腹 +30、体力 +10、経験値 +5 |
| おふろ | 清潔 +30、経験値 +5 |
| 散歩 | 体力 -15、満腹 -10、気分 +15、経験値 +8、猫との出会い判定 |
| 遊ぶ | 体力 -10、気分 +道具ごとの量、経験値 +8、道具入手判定 |

- 散歩は体力15未満、遊ぶは体力10未満のとき実行不可。「つかれてるみたい…」とメッセージ表示
- 各お世話後、メッセージエリアに反応メッセージを表示（複数パターンからランダム）

### 時間経過（ゲームを開いている間のみ）

- 20秒ごとに 満腹 -1、清潔 -1、体力 +2（各上限・下限あり）
- 満腹または清潔が30未満の間は気分も少しずつ下がる
- ブラウザを閉じている間は何も変化しない（放置ペナルティなし）

### レベルと経験値

- 経験値バーが満タン（レベル×20）でレベルアップ、余りは持ち越し
- レベルアップ時にお祝いメッセージを表示
- レベルが上がると散歩での猫出会い確率が少し上がる（基本25% + レベル×2%、上限50%）

### 遊び道具

| 道具 | 気分UP | 入手方法 |
|---|---|---|
| 毛糸玉 | +15 | 最初から所持 |
| ねこじゃらし | +20 | 遊んだとき30%で入手（未所持のものから抽選） |
| ボール | +20 | 同上 |
| ぬいぐるみ | +25 | 同上 |

- 「遊ぶ」を押すと所持している道具を選ぶ小窓（モーダル）が開き、選んだ道具で遊ぶ
- 新しい道具を拾ったときは入手メッセージを表示

### ねこ図鑑（全6匹）

| No | 猫 | 画像 |
|---|---|---|
| 1 | ミルク（メインの白い猫） | main_cat.png（最初から登録済み） |
| 2 | グレーのトラ猫 | gray_tabby_cat.png |
| 3 | まっしろのもふもふ猫 | AI生成 |
| 4 | 茶トラ | AI生成 |
| 5 | 黒猫 | AI生成 |
| 6 | 灰色と白のハチワレ | AI生成 |

- 散歩で出会い判定に成功すると、未発見の猫から1匹ランダムに発見
- 発見時は「あたらしい猫に出会った！」ポップアップ（猫の画像と名前入り）＋図鑑登録
- 未発見の猫は hidden_cat_silhouette.png ＋「？？？」で表示
- ホーム画面下部の図鑑プレビューとモーダル図鑑の両方に反映
- 全6匹発見でお祝いメッセージ
- モーダル内の説明文「散歩や遊びで出会った猫が登録されます」は「散歩で出会った猫が登録されます」に修正

### その他の機能

- 名前変更: 「変更」ボタンで入力ダイアログ（prompt）。空入力なら変更しない。上限8文字
- ホームボタン: ページの一番上へスムーズスクロール
- 設定ボタン: 「はじめからやり直す？」確認ダイアログ → OKでデータリセット
- セーブ: 状態変化のたびに localStorage へ自動保存。起動時に読み込み

## 技術方針

- 素のJavaScript 1ファイル（js/game.js）。ビルドツール・ライブラリなし（既存構成を踏襲）
- 状態は単一の state オブジェクトで管理し、render 関数で画面へ反映
- HTMLへの追加: 道具選択モーダルと猫発見ポップアップ（既存 .modal のスタイルを流用）
- フォルダ名 `CSS` → `css` に修正（公開時の互換性のため）
- assets/ は index.html が参照する構成（cats / actions / ui）で配置

## 画像生成

- nano-banana（画像生成AI）で既存画像と同じテイスト（透過PNG・手描き風）の画像を生成
  - 猫4匹: まっしろのもふもふ猫・茶トラ・黒猫・灰色と白のハチワレ
  - 道具3つ: ねこじゃらし・ボール・ねずみのおもちゃ
- 生成した画像はユーザー（娘さん）に確認してもらってから採用する

## 追記: プレイフィードバック対応（2026-07-04）

娘さんが実際に遊んだ結果のフィードバック3件に対応する。

### (1) メッセージがクリップの飾りと重なる表示バグ

`.memo-card::before`（クリップの輪っか）がカード上部に約21px分めり込んでおり、その直下から `.memo-card p`（メッセージ文）が始まるため重なる。`.memo-card p` に `padding-top` を追加してクリップの下に隠れないようにする。

### (2) 清潔メーターが減らない

現状は時間経過（tick）でのみ -1 ずつ減る仕様で体感しにくいため、お世話操作でも減るようにする。

- ごはん: 清潔 -5（食べこぼしイメージ）
- 散歩: 清潔 -10（外で汚れる）
- 遊ぶ: 清潔 -8（じゃれて汚れる）
- おふろは引き続き清潔 +30 のみ

### (3) メイン表示の猫を交換できる機能

図鑑（ホーム画面のプレビュー・モーダルどちらも）で、発見済みの猫をタップすると「（猫の名前）をメインのねこにする？」と確認ダイアログが出て、OKで画面中央に表示される猫（メイン猫）が切り替わる。

- 状態に `mainCatId`（初期値 `"milk"`）を追加し、中央の猫画像はこの値から表示する
- 現在メインになっている猫は図鑑内で枠を強調表示する
- 未発見（シルエット表示）の猫はタップしても反応しない

## 追記: タブレット・PC向けレイアウト（2026-07-04）

現状は `.phone-screen` が `width: min(100vw, 430px)` でスマホ縦長固定のため、タブレット・PCで開くと中央に小さく表示され左右の余白が大きい。ビジュアルコンパニオンで3案を比較し、以下の方針で決定。

- **画面幅768px未満（スマホ）**: 現状の縦並びレイアウトのまま変更しない
- **画面幅768px以上（タブレット・PC）**: 3カラムのダッシュボード型レイアウトに切り替える
  - 左列: ねこ図鑑（プレビュー）
  - 中央列: 猫の画像＋満腹・清潔・体力メーター
  - 右列: なまえ・レベル・気分のステータスカード＋お世話ボタン5つ
- 下部ナビ（ホーム・毛糸飾り・設定）は幅いっぱいの下部に据え置き
- 実装方針: HTMLの要素の並び順・IDは変更しない。`@media (min-width: 768px)` 内で親要素を CSS Grid 化し、各セクションに `grid-column` / `grid-row` を指定して視覚的な並びだけを変更する。JavaScript側の変更は不要
- モーダル（図鑑・道具選択・猫発見）は画面幅にかかわらず現状のまま中央オーバーレイ表示

## 追記: 画像フォーマット変更と猫ごとの見た目・進捗管理（2026-07-04）

Codexで画像を追加生成した際に、あわせて仕様も見直された。

### 画像フォーマットと配置

- 画像は PNG から WebP に変更し、既存の `assets/` に代わって以下の構成を使う
  - `assets_optimized/` : 通常時に表示する猫・お世話アイコン・UIアイコン（1枚ずつ）
  - `assets_variants/` : 猫ごとの表情・成長バリエーション画像（1匹につき5枚）
- 旧 `assets/`（PNG）はどこからも参照されなくなったが、削除はしていない

### 猫ごとの見た目バリエーション

各猫（`CATS`配列の各要素）に `variants` として以下5枚の画像を持たせる。

| variant | 用途 |
|---|---|
| `kitten` | レベル3未満（子猫）の通常表示 |
| `kittenHappy` | 子猫がお世話直後（3.5秒間）に見せる表情 |
| `kittenSad` | 子猫が満腹・清潔・体力のいずれか30未満、または気分が低いときの表情 |
| `happy` | レベル3以上（成猫）がお世話直後（3.5秒間）に見せる表情 |
| `sad` | 成猫が満腹・清潔・体力のいずれか30未満、または気分が低いときの表情 |

表示する画像の優先順位は「子猫かどうか」→「しょんぼり状態かどうか」→「お世話直後（うれしい状態）かどうか」→通常画像、の順で決まる。

### 猫ごとのレベル・経験値管理

- 以前はレベル・経験値がゲーム全体で1つ（グローバル）だったが、`catProgress`（猫IDごとのレベル・経験値マップ）に変更し、猫ごとに個別に育つようにした
- 新しく発見した猫・メインに設定した猫は、初回アクセス時に自動でレベル1・経験値0から開始する
- 子猫（レベル3未満）→成猫（レベル3以上）の切り替わりは、その猫自身のレベルで判定する

### 気分がしょんぼりになる条件の拡張

- 従来は「満腹・清潔のいずれかが30未満」のときだけ気分が下がっていたが、「体力（energy）が30未満」の場合も対象に加わった
- 散歩・遊びは気分を上げる行動だが体力を消費するため、直後に体力が30を切っていると次の時間経過（20秒ごと）で気分が下がることがある。意図した仕様として扱う

### 実装上の注意点（レビューで判明し修正済み）

- 猫ごとの育成管理に移行する際、旧セーブデータ（レベル・経験値がグローバルだった時代のデータ）は「保存当時メインだった猫」の進捗として引き継ぐ（固定の猫IDに紐付けない）
- 「お世話直後のうれしい反応」は、その反応を起こした猫がメインのままである間だけ表示する。メイン猫を切り替えたり、データをリセットした場合は反応をリセットする
